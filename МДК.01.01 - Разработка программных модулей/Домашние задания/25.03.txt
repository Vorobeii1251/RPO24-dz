1. Общее понятие
Модификаторы доступа определяют, где можно обращаться к членам класса (полям и методам).

Цель: Ограничение доступа, защита данных, реализация инкапсуляции.

2. Три модификатора
Модификатор	Доступ внутри класса	Доступ в производных классах	Доступ вне класса (через объект)
public	✅ Да	✅ Да	✅ Да
protected	✅ Да	✅ Да	❌ Нет
private	✅ Да	❌ Нет	❌ Нет
3. Подробно о каждом
public (открытый):

Доступен везде.

Используется для интерфейса класса (методы, которыми пользователь обязан уметь пользоваться).

private (закрытый):

Доступен только внутри самого класса.

Через объект напрямую обратиться нельзя.

Служит для защиты данных от некорректного изменения.

protected (защищенный):

Недоступен извне (как private).

Доступен внутри и в классах-наследниках.

Нужен, когда наследники должны иметь прямой доступ к данным родителя, а внешний мир — нет.

4. Инкапсуляция (главный принцип)
Определение: Сокрытие внутренних данных и предоставление доступа к ним только через специальные методы (геттеры/сеттеры).

Как реализуется: Данные делают private, а публичные методы (public) управляют ими (например, проверяют корректность значений).

Плюсы: Надежность, контроль изменений, защита от "дурака".
1. Что такое метод класса?
Функция внутри класса для работы с его данными.

Цели: управление состоянием, поведение объекта, инкапсуляция.

2. Где определять методы?
Внутри класса: автоматически становятся inline (хорошо для небольших функций).

Вне класса: объявляем внутри (void start();), реализуем снаружи через ИмяКласса:: (оператор разрешения области видимости).

3. Параметры и возврат
Методы работают как обычные функции: могут принимать параметры и возвращать значения.

4. Указатель this
Скрытый параметр каждого нестатического метода.

Указывает на текущий объект.

Пример: this->name = name; (поле = параметр).

5. Стандартные методы доступа
Getter – возвращает значение поля (для чтения).

Setter – изменяет значение поля (для записи и контроля).

6. Преимущества
Инкапсуляция и контроль доступа.

Удобство и повторное использование кода.

7. Правила вызова
Метод объявляется в классе.

Вызывается через объект: объект.метод().

Внутри метода доступен this.
