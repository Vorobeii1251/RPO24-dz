Краткий конспект: компиляция и сборка проекта
Зачем понимать процесс компиляции
В маленьких проектах это не критично.
При нескольких файлах, подключении библиотек и непонятных ошибках — необходимо.
Что такое компиляция
Процесс состоит из четырёх этапов:

Препроцессинг — обработка директив (#include, #define, #ifdef).
Компиляция — проверка синтаксиса и типов, генерация объектного кода (.o, .obj).
Ассемблирование — преобразование объектного кода в машинные инструкции (обычно автоматически).
Линковка — объединение объектных файлов, поиск реализаций функций, подключение библиотек.
Зачем нужны .h и .cpp файлы
.h — объявления, интерфейс.
.cpp — реализация.
Если всё писать в одном .cpp:

Сложно поддерживать.
Легко получить дублирование.
Линковка выдаст ошибки.
Типичные ошибки линковки
undefined reference (нет реализации функции).
Два определения одной функции.
Debug vs Release
Режим	Скорость	Проверки	Отладка
Debug	Медленно	Много	Удобно
Release	Быстро	Мало	Сложнее
Флаги компилятора
Используются для управления оптимизацией, включением отладочной информации и другими параметрами.

Почему IDE всё скрывает
Автоматически вызывает компилятор и линковщик.
Добавляет нужные флаги.
Минимальная сборка вручную (g++)
Пример базовой команды для компиляции и сборки проекта.

Статические и динамические библиотеки
.a / .lib — статические (подключаются на этапе линковки).
.so / .dll — динамические (подключаются на этапе линковки или загрузки).
